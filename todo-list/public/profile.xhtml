To-Do List API Profile

This document acts as the To-Do List API's profile, describing its application semantics.

The API allows for the programmatic management of tasks.

Vocabulary

Terms used in this API are defined below using XMDP. Here are a few XPath expressions for utilizing this vocabulary programmatically:

  • //dl[@class='profile']/dt - All defined terms
  • //dl[@class='profile']/dt[@id='{term}']/following-sibling::dd[1] - Definition of a given term
  • //dl[@class='profile']/dt[@id='status']/following-sibling::dd[1]/dl/dt - All possible values for the term status
collection
When used as a link relation type, its definition matches the one in Section 2.2 of RFC 6573. When used as a representation classifier, describes a collection of items.
complete
Boolean flag indicating whether a task has been completed.
create
Identifies the hypermedia control for creating a task.
edit
Identifies the hypermedia control for updating a task. Completed tasks cannot be updated.
delete
Identifies the hypermedia control for deleting a task.
task
Work to be done.
toggle-complete
Identifies the hypermedia control for toggling a task's complete flag.
title
Human-readable text labeling a task.

The following link relations are borrowed from the IANA registry:

  • collection
  • item
  • next
  • previous
  • self
